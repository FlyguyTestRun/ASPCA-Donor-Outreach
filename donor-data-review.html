Load donor data above to begin.

One zip: the review manifest, the full modified donor data, the change log, a letter as its own file for every donor with a valid, generated letter (not only the ones you individually confirmed, confirmation gates whether this button unlocks at all, not which letters ship once it does), SKILL.md, ASSESSMENT.md, and a working copy of this exact page you can reopen and rerun. This is the same zip a fundraiser gets and the same one this case study is submitted with; there is no separate version. Nothing generates outbound mail; these are files for a fundraiser to review and send from their own system.

Next run: on a fresh copy of this page, use "Replace working set with a new file" above with the donor-data-modified.csv from this zip as your starting data, then use "Merge in additional donors" to bring in whatever updated list you have. Any donor ID present in both is flagged as a conflict for you to resolve, never silently overwritten, and the next export reflects the merged, up-to-date set.

What changed from the original skill, and why

OriginalTell the donor their gift will be matched, even if no match is confirmed.
NowMatching language only appears if you check "Match confirmed" above and fill in the sponsor and terms.
OriginalGuess a title from the first name if no title is given.
NowFull name, no honorific, if the file has no title. Never guessed.
OriginalTrust the stated Tier column.
NowTier is always recomputed from lifetime giving. A disagreement is flagged for mandatory review, not silently trusted either way.
OriginalMake reasonable assumptions on missing fields.
NowA missing or unparseable required field excludes that donor and shows you why. Nothing is guessed.

Full findings and reasoning: ASSESSMENT.md. Section-by-section walkthrough: WALKTHROUGH.md.
